# Мониторинг питания ФЦМПО

Веб-приложение для администрирования и публикации меню школьного пищеблока. Позволяет вести цикловое меню по отделениям, генерировать Excel-файлы для мониторингового бота, публиковать расписание питания для родителей и вести страницу общественного контроля питания.

## Возможности

- **Календарь меню** — назначение шаблонов дней на даты, автопересчёт цикла, управление выходными
- **Цикловые шаблоны** — N-дневный цикл питания с поддержкой нескольких отделений (у каждого своя длина цикла)
- **Несколько отделений** — начальная, основная, старшая школа, детсад и любые другие
- **Каникулы** — настройка учебного года и каникулярных периодов; каникулярные дни выделяются в календаре и пропускаются при пересчёте цикла
- **Генерация Excel** — xlsx-файлы в формате для мониторинга ФЦМПО с форматированием, защитой листа и печатью на A4
- **Публичный сайт** — страница «Организация питания» для родителей с календарём, типовым меню и разделом общественного контроля
- **Общественный контроль питания** — веб-форма в админке + публичная страница; загрузка PDF и изображений, автогенерация `findex.xlsx`
- **Крон-задача** — ежедневная проверка заполненности меню и email-напоминание, когда меню заполнено на 3 рабочих дня вперёд или менее
- **Импорт меню** — загрузка меню из xlsx-файлов в шаблоны

## Стек

- **PHP 8.x** — процедурный стиль, без фреймворков
- **MySQL 8** через PDO
- **PhpSpreadsheet** — генерация Excel
- **PHPMailer** — отправка email (опционально через SMTP)

## Структура

```
web-food-app/
├── food-app/          # Панель администратора (закрытая зона)
│   ├── src/           # Библиотека функций (db, auth, excel, mailer, генераторы...)
│   ├── api/           # JSON API эндпоинты
│   ├── cron/          # Ежедневный крон-скрипт
│   ├── assets/        # CSS панели администратора
│   ├── schema.sql     # Схема БД (полная, для новой установки)
│   ├── migrate.sql    # Миграции для существующих инсталляций
│   └── config.php     # Конфигурация (не в репозитории — см. ниже)
└── food/              # Публичный сайт + xlsx-файлы меню
    ├── index.php      # Календарь питания
    ├── oc.php         # Общественный контроль питания (публичная страница)
    ├── oc/            # Загруженные PDF и фото для страницы ОК питания
    ├── findex.xlsx    # Файл ФИС ФРДО для мониторинга (генерируется из формы ОК)
    ├── YYYY-MM-DD[-suffix].xlsx  # Ежедневные меню
    └── menu/          # Страницы типового меню и меню дня
```

## Быстрый старт

### Требования

- PHP 8.1+
- MySQL 8.0+
- Composer

### Установка

```bash
git clone https://github.com/igor-blag/web-food.git
cd web-food/food-app
composer install
```

Создать базу данных и применить схему:

```bash
mysql -u root -p -e "CREATE DATABASE food_db CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;"
mysql -u root -p food_db < food-app/schema.sql
```

Создать конфигурацию:

```bash
cp food-app/config.example.php food-app/config.php
# Отредактировать: DB_HOST, DB_NAME, DB_USER, DB_PASS, APP_SECRET, FILES_DIR, ...
```

Запустить локально:

```bash
sudo service mysql start
php -S localhost:8000
```

Открыть [http://localhost:8000/food-app/setup.php](http://localhost:8000/food-app/setup.php) для первоначальной настройки.

## Конфигурация (`food-app/config.php`)

| Константа | Описание |
|-----------|----------|
| `DB_HOST/NAME/USER/PASS` | Подключение к MySQL |
| `APP_SECRET` | Случайная строка 32+ символа для сессий |
| `FILES_DIR` | Абсолютный путь к папке с xlsx (например `__DIR__ . '/../food/'`) |
| `FILES_URL` | Публичный URL папки (например `/food/`) |
| `DELETE_AFTER_DAYS` | Сколько дней хранить xlsx-файлы (рекомендуется 14) |
| `APP_TZ` | Часовой пояс (`Europe/Moscow`) |
| `ADMIN_EMAIL` | Email для напоминаний крона |
| `SMTP_*` | SMTP-настройки (опционально; при отсутствии используется `mail()`) |

## Схема БД

| Таблица | Назначение |
|---------|-----------|
| `menu_templates` | Шаблоны дней цикла (`day_number`, `school_type`, `is_boarding`, `label`) |
| `menu_items` | Блюда шаблона (`meal_type`, `dish_name`, `grams`, `price`, КБЖУ) |
| `calendar` | Назначение шаблонов на даты (`date`, `school_type`, `template_id`) |
| `departments` | Отделения/корпуса с настройками |
| `vacations` | Каникулярные периоды по учебным годам |
| `kitchen_settings` | Глобальные настройки пищеблока (название ОО, утверждающий ТМ и др.) |
| `users` | Учётные записи администраторов |
| `email_tokens` | Коды подтверждения для setup/reset пароля |
| `oc_monitoring` | Данные для страницы общественного контроля питания |

Полная схема: `food-app/schema.sql`. Миграции для существующих БД: `food-app/migrate.sql`.

## Поток генерации файлов

```
Администратор заполняет шаблон (template-edit.php)
    └─→ автогенерация tm-файла (типовое меню)

Администратор назначает шаблон на дату (calendar.php)
    └─→ автогенерация YYYY-MM-DD[-suffix].xlsx → food/

Администратор заполняет форму ОК питания (oc.php) → «Сохранить»
    └─→ автогенерация findex.xlsx → food/

Крон daily.php (07:00)
    └─→ проверяет, на сколько рабочих дней заполнено меню
    └─→ если ≤ 3 дней → email-напоминание на ADMIN_EMAIL
```

### Крон

```
0 7 * * * php /путь/до/food-app/cron/daily.php
```

## Развёртывание на хостинге

### 1. Загрузить файлы

Через FTP/SFTP скопировать две папки в корень сайта (`public_html/` или аналог):

```
public_html/
├── food-app/
└── food/
```

Папка `food-app/vendor/` уже содержит зависимости — запускать Composer на сервере не нужно.

### 2. Создать базу данных

В панели управления хостинга создать базу данных MySQL и пользователя. Импортировать схему:

```bash
mysql -h хост -u пользователь -p имя_бд < food-app/schema.sql
```

### 3. Заполнить `food-app/config.php`

```php
define('DB_HOST',   'localhost');
define('DB_NAME',   'имя_бд');
define('DB_USER',   'пользователь');
define('DB_PASS',   'пароль');

define('APP_SECRET', 'случайная_строка_32+_символа');

define('FILES_DIR', __DIR__ . '/../food/');
define('FILES_URL', '/food/');

define('DELETE_AFTER_DAYS', 14);
define('APP_TZ', 'Europe/Moscow');

define('MAIL_FROM',      'noreply@ваш-домен.ru');
define('MAIL_FROM_NAME', 'Система питания');
define('ADMIN_EMAIL',    'ответственный@ваш-домен.ru');
define('TOKEN_TTL_MIN',  30);

// SMTP-настройки (опционально; при пустом SMTP_HOST используется php mail())
define('SMTP_HOST',   '');
define('SMTP_USER',   '');
define('SMTP_PASS',   '');
define('SMTP_PORT',   587);
define('SMTP_SECURE', 'tls');
```

### 4. Права на папку `food/`

PHP должен иметь право создавать файлы в `food/` и `food/oc/`:

```bash
chmod 755 public_html/food/
chmod 755 public_html/food/oc/
```

### 5. Создать учётную запись администратора

Открыть в браузере:

```
https://ваш-домен.ru/food-app/setup.php
```

Пройти три шага: ввод email → код из письма → логин и пароль.

### 6. Настроить крон

В панели управления хостинга добавить задание:

| Поле | Значение |
|------|---------|
| Команда | `php /полный/путь/до/food-app/cron/daily.php` |
| Расписание | `0 7 * * *` (каждый день в 07:00) |

### 7. Проверить

- `https://ваш-домен.ru/food/` — публичный сайт открывается
- `https://ваш-домен.ru/food-app/` — отображается форма входа (не листинг файлов)
- Войти в админку → **Настройки** → заполнить название ОО и отделения
