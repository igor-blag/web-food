# food-app — панель администратора

Закрытая зона управления меню. Доступ только после авторизации.
`.htaccess` запрещает публичный доступ к папке.

## Страницы

### Авторизация и настройка

| Файл | Назначение |
|------|-----------|
| `index.php` | Форма входа. Если пользователей нет — редирект на `setup.php` |
| `setup.php` | Первоначальная настройка: 3 шага — ввод email → код подтверждения → создание логина/пароля. Доступен только если таблица `users` пуста |
| `forgot-password.php` | Сброс пароля через 6-значный email-код. Логика: запрос email → отправка кода → ввод кода + нового пароля |
| `logout.php` | Уничтожение сессии, редирект на `index.php` |
| `settings.php` | Настройки пищеблока: название ОО, отделения, каникулы, параметры типового меню |

### Шаблоны меню

| Файл | Назначение |
|------|-----------|
| `templates.php?type=sm\|main\|ss` | Список шаблонов N-дневного цикла. Добавление/удаление дней. Показывает количество блюд по приёмам пищи. Ссылка на скачивание tm-файла |
| `template-edit.php?id=X` | Редактор блюд шаблона. Таблицы по приёмам пищи (Завтрак, Завтрак 2, Обед; для интернатов + Полдник, Ужин, Ужин 2). Флаг `is_boarding` управляет видимостью интернатских секций. При сохранении sm-шаблона автоматически пересоздаётся `tm{year}-sm.xlsx` |
| `import.php?id=X` | Импорт меню из Excel в шаблон. Загрузка файла → парсинг через `src/importer.php` → перезапись блюд |
| `bulk-import.php` | Массовый импорт нескольких файлов Excel |

### Календарь

| Файл | Назначение |
|------|-----------|
| `calendar.php?type=&y=&m=` | Основной рабочий экран. Сетка месяца + боковая панель. Чекбокс на ячейке = «учебный день». Изменение чекбокса вызывает JS-пересчёт цикла и отправку `bulk_save` в API. Кнопки «Сбросить и пересчитать» / «Пересчитать» управляют расстановкой шаблонов цикла |
| `day-edit.php?date=&type=` | Детальный редактор конкретного дня. Выбор шаблона, школы, корпуса. При сохранении с шаблоном автоматически генерирует xlsx |

### Общественный контроль питания

| Файл | Назначение |
|------|-----------|
| `oc.php` | Форма заполнения 7 разделов общественного контроля. Поддерживает загрузку PDF и изображений в `food/oc/`. Изображения оптимизируются (max 2000 px, JPEG 82%). При нажатии «Сохранить» данные записываются в БД и автоматически генерируется `food/findex.xlsx` |

### Генерация документов

| Файл | Назначение |
|------|-----------|
| `kp.php?year=&type=` | Генерирует `kp{year}-{type}.xlsx` (годовой план питания — сетка-календарь с номерами дней цикла) и сразу отдаёт браузеру для скачивания |

## Зависимые папки

| Папка | Содержимое |
|-------|-----------|
| `src/` | Библиотека функций (DB, авторизация, генераторы, парсер, почта) |

## Конфигурация (`config.php`)

Все настройки в одном файле:

```php
DB_HOST, DB_NAME, DB_USER, DB_PASS  // MySQL
APP_SECRET                           // Ключ сессии (32+ случайных символа)
FILES_DIR                            // Абсолютный путь к папке xlsx → '../food/'
FILES_URL                            // Публичный URL папки → '/food/'
DELETE_AFTER_DAYS                    // Срок хранения xlsx (14 дней)
APP_TZ                               // Часовой пояс ('Europe/Moscow')
MAIL_FROM, MAIL_FROM_NAME            // Отправитель email
ADMIN_EMAIL                          // Получатель напоминаний (пусто = не отправлять)
TOKEN_TTL_MIN                        // Срок действия кода подтверждения (30 минут)
SMTP_HOST/USER/PASS/PORT/SECURE      // SMTP (необязательно; пусто = PHP mail())
```

## Авторизация

Сессионная, однопользовательская. Куки: `menu_sess`.
- `require_auth()` в начале каждой защищённой страницы — редирект на `index.php` если нет сессии
- `session_regenerate_id(true)` при входе (защита от session fixation)

## Логика работы JS в `calendar.php`

Весь AJAX — через `api/save-day.php`.

1. Пользователь ставит/снимает чекбокс на ячейке
2. JS собирает все отмеченные ячейки месяца в порядке дат
3. Распределяет номера шаблонов цикла начиная с `cycle-start-num`
4. Строит массив `days[]` для `bulk_save` (включая явные выходные `day_num=0` и удаления `day_num=-1`)
5. Отправляет POST на `api/save-day.php?action=bulk_save`
6. Обновляет DOM без перезагрузки страницы

Настройка «Рабочие дни» хранится в `localStorage` под ключом `defaultWorkdays_{type}`.
