# cron/ — автоматические задачи

## `daily.php`

Cron-скрипт ежедневного обслуживания. Защищён от HTTP-запросов — выполняется только из CLI (`php_sapi_name() !== 'cli'` → 403).

### Настройка на хостинге (Jino)

```
Команда:   php /home/LOGIN/public_html/food-app/cron/daily.php
Расписание: 0 7 * * *   (каждый день в 07:00)
```

### Что делает

**1. Генерация Excel-файлов (7 дней × 3 типа школ)**

Перебирает даты с сегодня по +6 дней вперёд, для каждой даты вызывает `generate_menu_excel($date, $type)` по всем типам (`sm`, `main`, `ss`).

Функция сама проверяет наличие шаблона в календаре — если день не учебный или шаблон не назначен, возвращает `null` и строка тихо пропускается.

Это перекрывает случай, когда администратор правил шаблон уже после ручной генерации файла.

**2. Очистка старых файлов**

Вызывает `cleanup_old_files()` из `src/excel.php` — удаляет `*.xlsx` из `FILES_DIR` старше `DELETE_AFTER_DAYS` дней (14 дней).

**3. Email-напоминание о незаполненных днях**

Работает только если `ADMIN_EMAIL` задан в `config.php`.

`find_missing_days($today, 3)` — проверяет ближайшие 3 рабочих дня (Пн–Пт): нет записи в `calendar` или `template_id IS NULL` → день считается незаполненным.

Если есть незаполненные дни → `send_reminder()` отправляет HTML-письмо с их списком.
