# cron/ — автоматические задачи

## `daily.php`

Крон-скрипт ежедневного мониторинга. Защищён от HTTP-запросов — выполняется только из CLI (`php_sapi_name() !== 'cli'` → 403).

### Настройка на хостинге (Jino)

```
Команда:    php /home/LOGIN/public_html/food-app/cron/daily.php
Расписание: 0 7 * * *   (каждый день в 07:00)
```

### Что делает

Считает, на сколько **последовательных** рабочих дней вперёд (начиная с завтра) заполнено меню для `school_type='sm'`. Подсчёт прерывается на первом незаполненном рабочем дне.

Если заполненных дней **3 или менее** — отправляет email-напоминание на `ADMIN_EMAIL`.

Если `ADMIN_EMAIL` не задан в `config.php` — письмо не отправляется, скрипт завершается молча.

### Пример вывода

```
[2026-03-13 07:00:01] Заполненных рабочих дней вперёд: 2
[OK] Напоминание отправлено на admin@school.ru
[2026-03-13 07:00:01] Завершено.
```

```
[2026-03-13 07:00:01] Заполненных рабочих дней вперёд: 7
[2026-03-13 07:00:01] Завершено.
```

### Логика подсчёта (`count_filled_workdays_ahead`)

1. Начинает с завтрашнего дня
2. Пропускает выходные (сб/вс) и каникулярные дни
3. Для каждого рабочего дня проверяет запись в `calendar` (`school_type='sm'`)
4. Если запись есть и `template_id IS NOT NULL` — день считается заполненным
5. При первом незаполненном рабочем дне — останавливается
6. Возвращает количество последовательно заполненных дней

### Содержимое email-напоминания

Тема: `Напоминание: меню заполнено на N дн.`

Текст: «Меню в системе мониторинга питания заполнено только на **N рабочих дня** вперёд. Пожалуйста, пополните расписание питания.»
