# src/ — библиотека функций

Все файлы подключаются через `require_once`. Нет автозагрузки — зависимости подключаются явно в каждом скрипте.

---

## `db.php` — база данных

**Singleton PDO-подключение** + все функции работы с БД.

```php
db() → PDO   // Возвращает единственный экземпляр соединения
```

### Функции шаблонов

| Функция | Что делает |
|---------|-----------|
| `get_templates(type)` | Все шаблоны типа школы, отсортированные по `day_number` |
| `get_template(id)` | Один шаблон по ID |
| `add_template(type)` | Создаёт новый шаблон со следующим `day_number` |
| `delete_template(id)` | Удаляет шаблон (блюда удаляются каскадом, в `calendar` → NULL) |
| `set_template_boarding(id, flag)` | Устанавливает флаг интерната |
| `get_templates_ordered(type)` | Возвращает `[day_number => template_id]` — нужен для JS и assign_cycle |

### Функции блюд

| Функция | Что делает |
|---------|-----------|
| `get_template_items(template_id)` | Блюда шаблона, сгруппированные по `meal_type` → `['breakfast'=>[...], 'lunch'=>[...], ...]` |
| `save_template_items(template_id, items[])` | Полная перезапись блюд шаблона (DELETE + INSERT) |

### Функции календаря

| Функция | Что делает |
|---------|-----------|
| `get_calendar_day(date, type)` | Запись на конкретную дату + JOIN с шаблоном |
| `get_calendar_month(year, month, type)` | Все записи месяца, ключ = `'Y-m-d'` |
| `get_calendar_year(year, type)` | Все записи года, ключ = `'Y-m-d'` |
| `save_calendar_day(date, tpl_id, school, dept, type, is_cycle_start)` | INSERT ... ON DUPLICATE KEY UPDATE |
| `delete_calendar_day(date, type)` | Удаляет запись (день становится «не учебным») |
| `get_cycle_length(type)` | Количество шаблонов (= длина цикла) |

### Функции массовых операций

| Функция | Что делает |
|---------|-----------|
| `bulk_save_calendar(days[], type)` | Массовое обновление календаря. `day_num=-1` → удалить, `0` → явный выходной, `N` → шаблон дня N |
| `assign_cycle(start_date, start_day, type, school, dept, end_date)` | Заполняет Пн–Пт от `start_date` до конца года (или `end_date`) шаблонами цикла. Пропускает явные выходные (записи с `template_id=null`). Возвращает количество назначенных дней |

## `auth.php` — авторизация

Сессионная авторизация. Зависит от `config.php` (APP_SECRET не используется напрямую — сессия PHP).

| Функция | Что делает |
|---------|-----------|
| `session_init()` | Запускает сессию с `httponly` куками и `strict_mode`. Имя сессии: `menu_sess` |
| `require_auth()` | Если нет `$_SESSION['user_id']` — редирект на `index.php` и `exit` |
| `try_login(username, password)` | `password_verify()` по хешу из БД. При успехе: `session_regenerate_id(true)`, заполняет `$_SESSION` |
| `do_logout()` | `session_destroy()` + редирект |
| `current_user()` | Возвращает `$_SESSION['username']` или null |

## `excel.php` — генератор меню дня

Генерирует xlsx-файл меню для конкретной даты и типа школы.

### `generate_menu_excel(date, type='sm') → ?string`

Возвращает путь к сохранённому файлу или `null` если день не учебный / шаблон не назначен.

**Структура xlsx:**
- Строка 1: шапка (Школа, Отд./корпус, Дата)
  - J1: дата как Excel Date-тип, формат `m/d/yyyy`
  - F1: формат `@` (текстовый, для корпуса)
- Строка 2: разделитель (высота 7.5)
- Строка 3: заголовки колонок A–J
- Строки 4+: блюда по секциям (Завтрак, Завтрак 2, Обед; интернат + Полдник, Ужин, Ужин 2)
  - Каждая секция заканчивается «закрывающей» строкой с жирной нижней границей
  - Колонка A (приём пищи): белый фон, только у первой строки секции верхняя жирная граница
  - Колонки B–J: жёлто-бежевый фон (`FFFFF2CC`)

**Имя файла:** `YYYY-MM-DD.xlsx` (main) или `YYYY-MM-DD-sm.xlsx` / `YYYY-MM-DD-ss.xlsx`

### `cleanup_old_files() → int`

Удаляет `*.xlsx` из `FILES_DIR` старше `DELETE_AFTER_DAYS` дней. Возвращает количество удалённых файлов.

## `kp_generator.php` — генератор плана питания

### `generate_kp_excel(year, type='sm') → string`

Генерирует `kp{year}-{type}.xlsx` — годовая сетка-календарь с номерами дней цикла.

**Структура xlsx:**
- Строка 1: Школа, заголовок «Календарь питания», год
- Строка 2: пустая
- Строка 3: заголовки — «Месяц» + числа 1–31
- Строки 4–13: 10 учебных месяцев (янв–июн, сен–дек)
  - Колонка A: название месяца
  - Колонки B–AF: номер дня цикла (жёлтый фон) или пусто (рабочий без шаблона)
  - Серый фон: несуществующие даты; светло-серый: выходные (сб/вс)

## `tm_generator.php` — генератор типового меню

### `generate_typical_menu_excel(school_type, year) → string`

Генерирует `tm{year}-{type}.xlsx` — типовое примерное меню по всем дням цикла.

**Структура xlsx (колонки A–L):**
- Строки 1–5: шапка (школа, заголовок, возрастная категория, год, заголовки колонок)
- Строки 6+: данные по шаблонам в порядке `day_number`
  - Для каждого шаблона: блок «Завтрак» (завтрак + завтрак2) + «Обед» + строки «итого» + «итого за день»
  - Колонки недели (A) и дня (B) в блоке обеда заполнены формулами-ссылками на блок завтрака

Вызывается автоматически при сохранении sm-шаблона в `template-edit.php`.

## `importer.php` — парсер xlsx

### `parse_menu_xlsx(filepath) → array`

Парсит xlsx-файл меню в массив, готовый для `save_template_items()`.

**Ожидаемая структура входного файла:**
- Строка 3: заголовки (игнорируются)
- Строки 4+: данные
  - Колонка A: название приёма пищи (только в первой строке секции: «Завтрак», «Обед» и т.д.)
  - Колонка B: раздел, C: № рецептуры, D: блюдо, E: выход (г), F: цена, G: ккал, H–J: Б/Ж/У

Маппинг русских названий в `meal_type`: «завтрак» → `breakfast`, «завтрак 2» → `breakfast2`, и т.д.

## `mailer.php` — почта

| Функция | Что делает |
|---------|-----------|
| `send_mail(to, subject, html)` | Отправляет письмо. Если `SMTP_HOST` задан — через SMTP (PHPMailer), иначе через `mail()` |
| `send_token(user_id, purpose, email)` | Генерирует 6-значный числовой код, инактивирует предыдущие коды того же purpose, сохраняет в `email_tokens`, отправляет письмо |
| `verify_token(user_id, purpose, code)` | Проверяет код: не истёк (`expires_at`), не использован, совпадает. При успехе — помечает как `used=1` |

`purpose` = `'setup'` (первоначальная настройка) или `'reset'` (сброс пароля).

## `header.php` — навигация

Подключается в начале body всех страниц администратора через `include`.
Выводит навигационную панель: логотип, ссылки (Календарь, Шаблоны, kp-файл), имя пользователя и кнопку выхода.
