# food/ — публичный сайт

Доступен без авторизации. Выполняет две роли:
1. **Сайт «Организация питания» для родителей** — календарь, типовое меню, общественный контроль питания
2. **Хранилище xlsx-файлов** — мониторинговый бот запрашивает файлы по URL `/food/YYYY-MM-DD[-suffix].xlsx` и `/food/findex.xlsx`

## Структура

```
food/
  .htaccess               ← Options -Indexes (запрещает листинг директории)
  index.php               ← Календарь питания (главная страница)
  oc.php                  ← Общественный контроль питания (публичная страница)
  findex.xlsx             ← Файл ФИС ФРДО (генерируется из формы ОК питания в админке)
  YYYY-MM-DD.xlsx         ← Меню дня (основная школа)
  YYYY-MM-DD-sm.xlsx      ← Меню дня (начальная школа)
  YYYY-MM-DD-ss.xlsx      ← Меню дня (старшая школа)
  tm{year}-sm.xlsx        ← Типовое меню (генерируется при сохранении шаблона)
  kp{year}-sm.xlsx        ← План питания (генерируется через kp.php)
  oc/
    .htaccess             ← Options -Indexes
    oc_*.pdf              ← Загруженные акты, приказы, материалы
    oc_*.jpg              ← Загруженные и оптимизированные фото
  menu/
    day.php               ← Меню на конкретный день
    menu.php              ← Типовое меню (все дни цикла с табами)
    assets/
      style.css           ← Стили публичного сайта
```

## Страницы

### `index.php` — календарь питания

Главная страница. Показывает только **текущий и будущие** месяцы (прошлые скрыты).

**Параметры URL:** `?type=sm|main|ss&y=YYYY&m=M`

- Логотип и заголовок берёт название ОО из `kitchen_settings.org_name`
- Табы переключения типа школы
- Навигация по месяцам
- Сетка: дни с назначенным меню — ссылки на `menu/day.php`, явные выходные — метка «Выходной»

### `oc.php` — общественный контроль питания

Публичная страница с семью разделами ФИС ФРДО:

1. Положение и приказ о создании комиссии
2. Формы взаимодействия с родителями (горячая линия, чат, форум)
3. Лечебные/диетические меню
4. Анкетирование
5. Информация о здоровом питании
6. Результаты контрольных мероприятий (акты + фото)
7. Оценка пищевых отходов

Данные берутся из таблицы `oc_monitoring`. Пустые разделы скрываются автоматически. PDF отображаются как кнопки скачивания, изображения — как превью.

### `menu/day.php` — меню дня

**Параметры URL:** `?date=YYYY-MM-DD&type=sm|main|ss`

Загружает запись из `calendar` + блюда из `menu_items`. Выводит таблицы по приёмам пищи. Для интернатских шаблонов (`is_boarding=1`) показывает расширенные секции (полдник, ужин).

### `menu/menu.php` — типовое меню

**Параметры URL:** `?type=sm|main|ss&day=ID`

Показывает все N дней цикла с табами. Данные из `menu_templates` + `menu_items`.

## Навигация

Все страницы имеют единую шапку:
- Логотип = название ОО из настроек (или «Организация питания» если не заполнено)
- Ссылки: Календарь | Типовое меню ▾ (dropdown по отделениям) | Общ. контроль

## Генерация xlsx-файлов

| Источник | Файл |
|----------|------|
| `food-app/calendar.php` (кнопка «Сохранить» в боковой панели) | `YYYY-MM-DD[-suffix].xlsx` |
| `food-app/day-edit.php` | `YYYY-MM-DD[-suffix].xlsx` |
| `food-app/template-edit.php` (сохранение sm-шаблона) | `tm{year}-sm.xlsx` |
| `food-app/kp.php` | `kp{year}-{type}.xlsx` |
| `food-app/oc.php` (кнопка «Сохранить» в форме ОК) | `findex.xlsx` |

Путь задан в `food-app/config.php`:
```php
define('FILES_DIR', __DIR__ . '/../food/');
define('FILES_URL', '/food/');
```

## Защита от листинга

`.htaccess` содержит `Options -Indexes`.

- Прямые GET-запросы к xlsx — Apache отдаёт как статику (мониторинговый бот работает)
- Запрос `GET /food/` — отвечает `index.php`
- PHP-файлы в `menu/` и корне — выполняются нормально
