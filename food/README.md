# food/ — публичный сайт

Доступен без авторизации. Выполняет две роли:
1. **Сайт для родителей** — просмотр расписания питания
2. **Хранилище xlsx-файлов** — мониторинговый бот запрашивает файлы по URL `/food/YYYY-MM-DD.xlsx`

## Структура

```
food/
  .htaccess          ← Options -Indexes (отключает листинг, PHP и xlsx работают)
  index.php          ← Публичный календарь (главная страница)
  YYYY-MM-DD.xlsx         ← Меню дня (основная школа)
  YYYY-MM-DD-sm.xlsx      ← Меню дня (начальная школа)
  YYYY-MM-DD-ss.xlsx      ← Меню дня (старшая школа)
  tm{year}-sm.xlsx        ← Типовое меню (генерируется при сохранении шаблона)
  kp{year}-sm.xlsx        ← План питания (генерируется через kp.php)
  menu/
    day.php          ← Меню на конкретный день
    menu.php         ← Типовое меню (все дни цикла с табами)
    assets/
      style.css      ← Стили публичного сайта
```

## Страницы

### `index.php` — календарь

Главная страница. Показывает только **текущий и будущие** месяцы (прошлые скрыты).

**Параметры URL:** `?type=sm|main|ss&y=YYYY&m=M`

- Табы переключения типа школы
- Навигация по месяцам (кнопки «← Предыдущий» / «Следующий →»)
- Сетка: дни с назначенным меню — ссылки на `menu/day.php`, явные выходные — метка «Выходной»
- Блок «Документы» (появляется если файлы существуют): ссылки на скачивание tm- и kp-файлов

Данные из БД берёт через `../food-app/config.php` и `../food-app/src/db.php`.

### `menu/day.php` — меню дня

**Параметры URL:** `?date=YYYY-MM-DD&type=sm|main|ss`

Загружает запись из `calendar` + блюда из `menu_items`. Выводит таблицы по приёмам пищи (Завтрак, Обед и т.д.). Для интернатских шаблонов (`is_boarding=1`) показывает расширенные секции.

Кнопка «↓ Скачать меню (.xlsx)» появляется если файл `FILES_DIR . $date . $suffix . '.xlsx'` существует.

Ссылка «← Назад к календарю» ведёт на `../index.php?type=...`.

### `menu/menu.php` — типовое меню

**Параметры URL:** `?type=sm|main|ss` (+ `?day=ID` при переключении табов)

Показывает все N дней цикла с табами. При выборе таба перезагружает страницу с `?day={template_id}`. Данные из `menu_templates` + `menu_items`.

## Генерация xlsx-файлов

Файлы создаются тремя способами:

| Источник | Когда |
|----------|-------|
| `food-app/day-edit.php` | При сохранении шаблона на конкретный день (автоматически) |
| `food-app/cron/daily.php` | Ежедневно в 07:00: 7 дней × 3 типа школ |
| `food-app/kp.php` | По запросу администратора (kp-файл) |
| `food-app/template-edit.php` | При сохранении sm-шаблона (tm-файл) |

Путь задан в `food-app/config.php`:
```php
define('FILES_DIR', __DIR__ . '/../food/');
define('FILES_URL', '/food/');
```

## Защита от листинга

`.htaccess` содержит `Options -Indexes`.

- Прямые запросы к xlsx (`GET /food/2026-03-04-sm.xlsx`) — работают, Apache отдаёт как статику
- Запрос `GET /food/` — возвращает `index.php` (не листинг директории)
- PHP-файлы в `menu/` — выполняются нормально
